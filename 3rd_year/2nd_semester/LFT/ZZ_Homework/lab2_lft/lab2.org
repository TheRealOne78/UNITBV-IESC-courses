** c) Ce token produce operatorul =->= ? Dar =!== ? Argumentați de ce nu sunt doi tokeni separați.

=->= produces =Arrow=. =!== produces =NotEq=.

Neither is split into two tokens because the lexer applies the *maximal munch* rule: it always consumes the longest character sequence that forms a valid token. When the lexer sees =-=, it peeks at the next character; finding =>= it consumes both and emits =Arrow=. The same logic applies to =!=: seeing ===, it emits =NotEq= instead of a bare =!= followed by ====.

* Exercițiul 2: Expresii regulate

** a) Expresia regulată pentru tipul float cu sufix opțional =f32= sau =f64=.

#+begin_example
[0-9]+\.[0-9]+(f32|f64)?
#+end_example

Examples: =3.14f32=, =2.0f64=, =1.5=

** b) Expresia regulată pentru un identificator Rust care nu poate începe cu două underscore consecutive.

#+begin_example
(?!__)[a-zA-Z_][a-zA-Z0-9_]*
#+end_example

The negative lookahead =(?!__)= rejects any identifier whose first two characters are both underscores. The rest of the pattern is the standard identifier rule.

** c) De ce șirul =12abc= nu poate fi tokenizat ca un singur token =INT=.

The regex for =INT= is =[0-9]+=, so the lexer stops as soon as it hits a non-digit. It reads =1= and =2=, then hits =a= which is not a digit, and emits =INT(12)=. It then tokenizes =abc= as =IDENT("abc")=. The result is two tokens, not one. This is consistent with maximal munch: =12= is the longest valid integer prefix, and =abc= is a separate valid identifier.

** d) DFA pentru operatorul de egalitate =Eq= (====), distinct de =Assign= (==).

#+begin_example
States:
  Start    -- initial state
  SeenEq   -- consumed one '='
  Assign   -- final: emit ASSIGN (next char was not '=')
  Eq       -- final: emit EQ (consumed second '=')

Transitions:
  Start  --['=']--> SeenEq
  SeenEq --['=']--> Eq       (second '=' → EQ)
  SeenEq --[other]--> Assign  (backtrack: current char not consumed)
#+end_example

** c) Suport pentru comentarii bloc nested =/* /* interior */ exterior */=.

Standard single-pass scanning cannot handle nested block comments because it stops at the first =*/= it finds, regardless of nesting depth. The fix is to track a depth counter and only close the comment when depth reaches zero.

Replace the =Some('*')= branch in =scan_slash()=:

#+begin_src rust
Some('*') => {
    self.advance(); // consume '*'
    let mut depth: usize = 1;
    loop {
        match self.peek() {
            None => {
                return self.report_error(
                    self.line, self.col,
                    "unterminated block comment", "/*"
                );
            }
            Some('/') if self.peek_next() == Some('*') => {
                self.advance(); self.advance();
                depth += 1;
            }
            Some('*') if self.peek_next() == Some('/') => {
                self.advance(); self.advance();
                depth -= 1;
                if depth == 0 { break; }
            }
            _ => { self.advance(); }
        }
    }
    return self.next_token().token;
}
#+end_src

* Exercițiul 4: Extinderea lexerului manual

** a) Suport pentru literale =char=: ='a'=, ='\n'=, ='\t'=.

In =token.rs=, add:

#+begin_src rust
CharLit(char),
#+end_src

In =next_token()=, add a new arm before the catch-all:

#+begin_src rust
Some('\'') => self.scan_char(),
#+end_src

Add the =scan_char= method to =impl Lexer=:

#+begin_src rust
fn scan_char(&mut self) -> Token {
    let start_line = self.line;
    let start_col  = self.col;
    self.advance(); // consume opening '\''

    let ch = match self.peek() {
        None | Some('\n') => {
            return self.report_error(start_line, start_col,
                "unterminated char literal", "'");
        }
        Some('\'') => {
            self.advance();
            return self.report_error(start_line, start_col,
                "empty char literal", "''");
        }
        Some('\\') => {
            self.advance();
            match self.peek() {
                Some('n')  => { self.advance(); '\n' }
                Some('t')  => { self.advance(); '\t' }
                Some('\\') => { self.advance(); '\\' }
                Some('\'') => { self.advance(); '\'' }
                Some(c) => {
                    self.advance();
                    return self.report_error(start_line, start_col,
                        &format!("unknown escape '\\{}'", c),
                        &format!("'\\{}'", c));
                }
                None => {
                    return self.report_error(start_line, start_col,
                        "unterminated char literal", "'\\");
                }
            }
        }
        Some(c) => { self.advance(); c }
    };

    match self.peek() {
        Some('\'') => { self.advance(); }
        _ => {
            return self.report_error(start_line, start_col,
                "unterminated char literal", &ch.to_string());
        }
    }
    Token::CharLit(ch)
}
#+end_src

Update =tokenize= to return the error list alongside the token list:

#+begin_src rust
pub fn tokenize(source: &str) -> (Vec<SpannedToken>, Vec<String>) {
    let mut lexer  = Lexer::new(source);
    let mut tokens = Vec::new();
    loop {
        let st     = lexer.next_token();
        let is_eof = st.token == Token::Eof;
        tokens.push(st);
        if is_eof { break; }
    }
    (tokens, lexer.errors)
}
#+end_src

    println!("Tokens");
    for st in &tokens {
        println!("{}:{} {:?}", st.line, st.col, st.token);
    }

    println!("\nErrors: ({})", errors.len());
    for e in &errors {
        println!("{}", e);
    }
}

* Exercițiul 5: Lexer cu logos — extindere

** a) Suport pentru comentarii bloc =/* ... */= ca token ignorat.

=token_logos.rs=:

#+begin_src rust
#[logos(skip r"/\*(?:[^*]|\*[^/])*\*/")]
#+end_src

This pattern matches =/\*=, then any sequence of chars that are either not =*=, or =*= not followed by =/=, and finally =\*/=. It handles single-level block comments; for nested comments a logos skip regex is insufficient and a callback would be needed.

** b) Suport pentru literale =char= în logos.

#+begin_src rust
#[regex(r"'([^'\\]|\\[ntr\\'])'", |lex| {
    let s     = lex.slice();
    let inner = &s[1..s.len() - 1]; // strip surrounding single quotes
    if inner.starts_with('\\') {
        match inner.chars().nth(1) {
            Some('n')  => Some('\n'),
            Some('t')  => Some('\t'),
            Some('\\') => Some('\\'),
            Some('\'') => Some('\''),
            _          => None,
        }
    } else {
        inner.chars().next()
    }
})]
CharLit(Option<char>),
#+end_src

The stored type is =Option<char>=: =Some(c)= for a valid character, =None= if the escape sequence is unrecognized.

** c) Comparație lexer manual vs. logos pe 1000 de linii.

Logos is by far much faster since it combines all token definitions into a single deterministic state machine, optimizes branches into lookup tables or jump tables, prevents backtracking inside token definitions, unwinds loops, and batches reads to minimize bounds checking and does all of that heavy lifting at compile time.
